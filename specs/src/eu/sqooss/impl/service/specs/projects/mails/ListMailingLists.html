Getting the Mailing Lists

In order to allow analysis involving the mail traffic within projects, the system should allow to explore projects mailing lists archives.

The system can create projects linked to a set of mailing list archives. The mailing lists can be listed for those projects.

Example

Given the system has a project named "savonet" which mailing list archives lie in the "../specs/data/savonet-mail/mail".

Then this project has 2 mailing lists. The following lists are available:
