Querying the Changes from a Revision

In order to allow analysis involving the code history, the system should allow to explore projects source control management.

The system can create projects linked to a source control repository. The revisions of the source code can be listed for those projects.

Example

Given the system has a project named "iCopy" which source repository lies in the "../specs/data/iCopy/svn".

Then the revision 38 has the following commit log:

New form for viewing/modifying scanner properties

And it contains the following changes:

File Type
iCopy/WIAProperties.Designer.vbadded
iCopy/WIAProperties.resxadded
iCopy/WIAProperties.vbadded

Also the revision 37 has the following commit log:

Added new strings

And it contains the following changes:

File Type
iCopy/WinFormStrings.Designer.vbmodified
iCopy/WinFormStrings.resxmodified
