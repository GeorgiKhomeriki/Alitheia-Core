Definition Groups Can't Have Users

The system allows to create groups and users, but it has a special type of groups, called "definition groups". They're used to partly define the security policy, but they don't accept to have users. Those groups should always be empty.

Example

At startup, the system has a group named web services service security group. This group is a definition group and is empty.

When we try to add the user sebas, the operation fails.

Then the group is still empty.
