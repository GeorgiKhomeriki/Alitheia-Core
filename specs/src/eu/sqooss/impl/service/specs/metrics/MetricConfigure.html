Metric Configure

Quality is meassured with different metric plugins. In order to use a plug-in, it needs to be installed.

Example

The system comes with registered, but not installed metric plug-ins.

After installing the "Productivity metric plug-in", the list becomes:

Plugin Name Installed
CLMT plug-in false
Productivity metric plug-in true
Wc metric plug-in false

Each metric plug-in can have properties. The default properties for the "Productivity metric plug-in" are:

Name Type Value
CMF_threshold INTEGER 5
Weights_Update_Interval INTEGER 150

After the value for the "CMF_threshold" property is changed "10", the properties value is now 10.

Three types of properties can be added: INTEGER, STRING and BOOLEAN.

After adding the "My Integer Property" property of type "INTEGER", with value "100", and

the "My String Property" property of type "STRING", with value "Test-string", and

the "My Boolean Property" property of type "BOOLEAN", with value "true", and:

Name Type Value
CMF_threshold INTEGER 10
My Boolean Property BOOLEAN true
My Integer Property INTEGER 100
My String Property STRING Test-string
Weights_Update_Interval INTEGER 150
